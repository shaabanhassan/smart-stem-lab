Chapter 01

Custom Blocks: Writing Your Own Instructions

"Professional programmers don't write the same code twice. When they need to do something more than once, they give it a name. That's called a function — and in Scratch, you can make your own."

By the end of this chapter, you will be able to:

  • Explain what a custom block is and why it's useful
  • Create a custom block in Scratch with and without inputs
  • Use custom blocks to make your code shorter and cleaner
  • Apply the DRY principle: Don't Repeat Yourself

What is a Custom Block?

In Scratch, you've been using blocks that came with the program — motion blocks, looks blocks, sound blocks. But what if you want a block that doesn't exist yet? What if you keep writing the same 5 blocks over and over again in your project?

Custom blocks let you create your own block with your own name. You write what it does once, give it a name, and then you can use that one block anywhere in your project — as many times as you like.

Key Concept: The DRY Principle

DRY = Don't Repeat Yourself. This is one of the most important rules in all of programming. If you find yourself writing the same group of blocks more than twice, that's a sign you need a custom block. Repeated code is harder to fix — if you make a mistake, you have to fix it in many places. A custom block means you only fix it once.

The Problem with Repeated Code

Imagine your game needs the player sprite to do a "jump" animation: rise up, pause, then fall back down. Every time the player presses the space bar, those same 4 blocks need to run. Without custom blocks, your code looks like this:

when [space] key pressed
change y by 30
wait 0.3 seconds
change y by -30
... and again when an enemy appears ...
when [touching enemy] detected
change y by 30
wait 0.3 seconds
change y by -30

That's the same 3 blocks written twice. In a real game, it might be written 10 times! Now imagine if you need to change the jump height. You'd have to change it in 10 different places. With a custom block called jump, you change it once.

How to Create a Custom Block

In Scratch, go to the My Blocks category (the pink category at the bottom). Click "Make a Block". Give your block a name. Click OK. A special new hat block appears — define [your block name] — and that's where you put the instructions for what your block does.

Teacher Tip: Block Naming Rules

Block names should be short and describe what they do. Good names: jump, shoot, explode, reset player. Bad names: block1, do the thing, aaaa. A good name tells anyone reading your code exactly what happens when that block runs.

A block you create yourself with your own name and instructions.

Chapter 02

Sprites Talking to Each Other

"In a movie, every actor has their own role — but they all know when to appear on screen because they're following a shared script. In Scratch, sprites work the same way. They broadcast messages, and other sprites listen and react."

By the end of this chapter, you will be able to:

  • Explain what broadcast and receive mean in Scratch
  • Make one sprite send a message that triggers another sprite
  • Use "broadcast and wait" to sequence multi-sprite actions
  • Build a scene with at least 3 sprites communicating

The Problem: Sprites Can't Talk Directly

Imagine your game has a player sprite and a coin sprite. When the player touches the coin, you want the coin to disappear AND the scoreboard sprite to add 10 points. How does the scoreboard know to update? The player touched the coin, not the scoreboard.

In Scratch, one sprite can't directly tell another sprite what to do. But it can broadcast a message — like shouting "Coin collected!" into the air — and any sprite listening for that message will react.

Real-World Analogy: Radio Broadcasting

Think of a radio station. The DJ broadcasts a song. Every radio in the city that's tuned to that station plays it at the same time. The DJ doesn't call each radio individually — they send one broadcast and everything listening reacts. Scratch's broadcast and receive works the same way.

How Broadcast and Receive Work

In the Events category, you'll find two important blocks:

To send a message:
broadcast [game over]
To receive a message (in any sprite):
when I receive [game over]
say [You lost!] for 2 seconds
stop [all]

The key thing to understand: any sprite in the project — including the sprite that broadcast the message — can listen for it and react. You can have 5 different sprites all react to the same broadcast at the same time.

Broadcast vs Broadcast and Wait

broadcast [message] — Sends the message and immediately continues to the next block without waiting.

broadcast [message] and wait — Sends the message and PAUSES until every receiving script has finished running. Use this when the order matters — when you need one thing to fully finish before the next thing starts.

Broadcast
Sending a message that all sprites can hear.
Receive
Listening for a broadcast message and running code when it arrives.
Message
A named signal sent between sprites.
Trigger
An event that causes code to run.
Parallel
Multiple scripts running at the same time.
Sequence
Scripts running one after another in order.

Chapter 03

Lists: A Backpack for Your Data

"A variable can hold one thing. A list can hold many things. Think of a variable as a pocket and a list as a backpack — you can carry a lot more."

By the end of this chapter, you will be able to:

  • Create a list in Scratch and understand what it stores
  • Add, delete, and read items from a list
  • Use a list to store quiz questions and answers
  • Loop through a list to use each item

What is a List?

You already know that a variable stores one piece of information at a time: score = 10, name = "Alex", lives = 3. But what if you need to store 10 questions for a quiz? Or 5 enemy positions? Or all the items in a player's inventory?

A list stores multiple items in a numbered sequence. The first item is at position 1, the second at position 2, and so on. You can add to a list, remove from it, read any item by its position number, and check how many items are in it.

Tip: Lists vs Variables

Use a variable when you need to store ONE thing that changes (like the score).
Use a list when you need to store MULTIPLE things (like all the scores on a leaderboard, or all the questions in a quiz).

Basic Robot Commands

Your robot kit (mBot or similar) can be programmed using a block-based interface (mBlock, which looks a lot like Scratch!) or with code. Here are the fundamental commands every robot program uses:

CommandWhat it doesImportant values
Move forward (speed)Both motors spin forward at a given speed (0–255)Speed 0 = stopped, 255 = maximum
Move backward (speed)Both motors spin backwardSame speed range
Turn left (speed)Right motor forward, left motor stopped (or reverse)Smaller speed = smaller turn

StopBoth motors stopAlways end your program with Stop!

If you forget to add a Stop command at the end of your program, your robot will keep moving forever after your last command runs — often driving off a table or hitting a wall. Always be the last instruction in your program.

How Timing Controls Distance

Your robot doesn't have a ruler. It can't count centimeters. Instead, you control how far it moves by controlling how fast it moves AND for how long. This is the key insight:

Distance = Speed × Time

If your robot moves at speed 150 for 2 seconds, it travels a certain distance. If you keep the speed the same but change to 4 seconds, it travels twice as far. You'll need to measure and test to find the right timing for your robot — every robot is slightly different!

The Ultrasonic Distance Sensor

Your robot kit likely has an ultrasonic sensor on the front — the two small circles that look like eyes. This sensor works exactly like a bat's echolocation. It sends out a sound wave (too high-pitched for humans to hear), the sound bounces off an obstacle, and the sensor measures how long it takes to come back. From that time, it calculates the distance to the obstacle in centimeters.

How Ultrasonic Sensing Works

Sound travels at about 343 meters per second. The sensor sends a pulse and starts a timer. When the echo returns, it stops the timer. Distance = (speed of sound × time) ÷ 2. We divide by 2 because the sound traveled to the obstacle AND back. Your robot does this calculation automatically — you just ask it: "how far is the nearest object?"

PowerPoint: One Idea Per Slide

"Most bad presentations make the same mistake: they put everything on every slide. The best presentations do the opposite — one clear idea, delivered simply, repeated confidently."

The Six Rules of Great Slides

Rule 1: One idea per slide
If you can say it in one sentence, put it on one slide. Don't cram.
Rule 2: No walls of text
Maximum 5 words per bullet point. Maximum 5 bullets per slide. Slides support your speech — they don't replace it.
Rule 3: Use contrast
Dark text on light background, or light text on dark background. Never gray text on white — it's unreadable.
Rule 4: Align everything
Use the alignment guides. Every text box and image should align to something. Random placement looks messy.
Rule 5: Images earn their place
Every image must explain something that words can't. If you're adding an image "for decoration," delete it.
Rule 6: Animations sparingly
One transition style for the whole deck. Simple animations only (Appear, Fade). Flying logos and spinning text = unprofessional.

Design Basics: Color, Shape & Space

Color: The Fastest Way to Communicate Feeling

Before a person reads a single word, they've already felt something from the colors on the page. Designers use this deliberately. Here's what research and tradition tell us about color and emotion:

Red
Energy, danger, urgency, passion. Used by: stop signs, fast food, alerts.
Orange
Warmth, creativity, excitement, playfulness.
Yellow
Happiness, optimism, caution. High contrast — very visible.
Green
Nature, health, money, go. Used by: environmental brands, banks, traffic lights.

Trust, calm, technology, professionalism. Most popular color for logos.
Purple
Royalty, mystery, creativity, luxury.

Positive and Negative Space

Positive space is where your content is — the shapes, the text, the images. Negative space is the empty area around and between the content. Most beginners try to fill every inch of the page. Expert designers do the opposite: they leave generous negative space to make the important elements stand out.

The FedEx Logo Secret

Look at the FedEx logo. Between the E and the X, there is a hidden arrow made entirely of negative space. You might have seen this logo a thousand times and never noticed. This is what designers call a "hidden image" in negative space. Once you see it, you can never unsee it. This is what awareness of negative space trains you to do: see the shapes that aren't there.

Understanding Layers

Think of layers like transparent glass sheets stacked on top of each other. You draw something on each sheet. When you look at all the sheets together from above, they combine into one complete picture. The magic is that each sheet is independent — you can move, change, or delete what's on one sheet without affecting the others.
